## Qué es y por qué existe

Apex estático resuelve el 80% de los casos. Dynamic Apex resuelve el 20% restante: cuando el tipo de objeto, los campos a consultar, o la lógica a ejecutar no se conocen hasta runtime. Es fundamental para frameworks reutilizables, integraciones genéricas y sistemas configurables sin deploy.

**Tres herramientas principales:**
1. **Dynamic SOQL** — queries construidos como strings en runtime
2. **Schema Describe** — introspección de metadata en runtime
3. **Dynamic DML + `Type.forName()`** — operar sobre objetos y clases desconocidos en compilación

## Dynamic SOQL

### Estático vs Dinámico

```apex
// Estático — compilado, validado por Salesforce al guardar
List<Contact> contacts = [SELECT Id, Email FROM Contact WHERE LastName = :lastName];

// Dinámico — string en runtime, Salesforce no lo valida al compilar
String query = 'SELECT Id, Email FROM Contact WHERE LastName = :lastName';
List<Contact> contacts = Database.query(query);
```

### Cuándo necesitas Dynamic SOQL

- Los campos a consultar los elige el usuario o vienen de configuración
- El objeto a consultar varía en runtime
- Las condiciones del WHERE varían dinámicamente
- Estás construyendo un framework o utilería genérica

### El riesgo: SOQL Injection

```apex
// VULNERABLE — si el usuario envía: Smith' OR Name != '
String query = 'SELECT Id FROM Contact WHERE LastName = \'' + userInput + '\'';
// Result: SELECT Id FROM Contact WHERE LastName = 'Smith' OR Name != ''
// → devuelve TODOS los contactos

// SEGURO — bind variables (preferido)
String lastName = userInput;
String query = 'SELECT Id FROM Contact WHERE LastName = :lastName';
// Salesforce escapa automáticamente

// SEGURO — escapeSingleQuotes cuando no puedes usar bind
String safeInput = String.escapeSingleQuotes(userInput);
String query = 'SELECT Id FROM Contact WHERE LastName = \'' + safeInput + '\'';
```

### El método moderno: `Database.queryWithBinds()`

```apex
Map<String, Object> binds = new Map<String, Object>{
    'lastName' => 'Smith',
    'isActive' => true
};

List<Contact> contacts = Database.queryWithBinds(
    'SELECT Id, Email FROM Contact WHERE LastName = :lastName AND Active__c = :isActive',
    binds,
    AccessLevel.USER_MODE  // respeta FLS y sharing del usuario actual
);
```

### `USER_MODE` vs `SYSTEM_MODE`

| | USER_MODE | SYSTEM_MODE |
|---|---|---|
| Respeta FLS | Sí | No |
| Respeta sharing | Sí | No |
| Cuándo usar | Queries desde UI | Procesos internos del sistema |

### Query Builder reutilizable y seguro

```apex
public class ContactQueryBuilder {

    private List<String> fields = new List<String>{'Id', 'Name'};
    private List<String> conditions = new List<String>();
    private Map<String, Object> binds = new Map<String, Object>();
    private Integer limitSize;

    public ContactQueryBuilder addField(String fieldName) {
        // Valida el campo contra el schema antes de agregarlo
        Map<String, Schema.SObjectField> contactFields =
            Schema.SObjectType.Contact.fields.getMap();
        if (contactFields.containsKey(fieldName.toLowerCase())) {
            fields.add(fieldName);
        }
        return this;
    }

    public ContactQueryBuilder whereEquals(String field, Object value) {
        String bindKey = 'bind_' + conditions.size();
        conditions.add(field + ' = :' + bindKey);
        binds.put(bindKey, value);
        return this;
    }

    public ContactQueryBuilder limitTo(Integer n) {
        this.limitSize = n;
        return this;
    }

    public List<Contact> execute() {
        String query = 'SELECT ' + String.join(fields, ', ') + ' FROM Contact';
        if (!conditions.isEmpty()) {
            query += ' WHERE ' + String.join(conditions, ' AND ');
        }
        if (limitSize != null) {
            query += ' LIMIT ' + limitSize;
        }
        return Database.queryWithBinds(query, binds, AccessLevel.USER_MODE);
    }
}

// Uso — patrón fluent/builder
List<Contact> results = new ContactQueryBuilder()
    .addField('Email')
    .addField('Phone')
    .whereEquals('LastName', 'Smith')
    .whereEquals('Active__c', true)
    .limitTo(50)
    .execute();
```

## Schema Describe — Introspección de Metadata

### Describir un SObject

```apex
Schema.DescribeSObjectResult describe = Contact.sObjectType.getDescribe();

String label       = describe.getLabel();
String labelPlural = describe.getLabelPlural();
Boolean isCreateable = describe.isCreateable();
String keyPrefix   = describe.getKeyPrefix(); // '003' para Contact
```

### Describir campos

```apex
Schema.DescribeFieldResult fieldDesc = Contact.Email.getDescribe();

String label         = fieldDesc.getLabel();
Boolean required     = !fieldDesc.isNillable();
Boolean editable     = fieldDesc.isUpdateable();
Integer maxLength    = fieldDesc.getLength();
Schema.DisplayType tipo = fieldDesc.getType(); // STRING, PICKLIST, REFERENCE, etc.

// Valores de un picklist
List<Schema.PicklistEntry> entries = fieldDesc.getPicklistValues();
for (Schema.PicklistEntry entry : entries) {
    String value  = entry.getValue();
    String label2 = entry.getLabel();
    Boolean active = entry.isActive();
}

// Objetos relacionados en un campo lookup
List<Schema.SObjectType> references = fieldDesc.getReferenceTo();
```

### Todos los campos de un objeto

```apex
Map<String, Schema.SObjectField> allFields =
    Schema.SObjectType.Contact.fields.getMap();

for (String fieldName : allFields.keySet()) {
    Schema.DescribeFieldResult f = allFields.get(fieldName).getDescribe();
    System.debug(fieldName + ' — ' + f.getType() + ' — ' + f.getLabel());
}
```

### `Schema.getGlobalDescribe()` — cachear obligatorio

```apex
// ES CARO — nunca llamar más de una vez sin cacheo
Map<String, Schema.SObjectType> allObjects = Schema.getGlobalDescribe();

// PATRÓN CORRECTO — Singleton con variable estática
public class SchemaCache {
    private static Map<String, Schema.SObjectType> globalDescribeCache;

    public static Schema.SObjectType getSObjectType(String objectName) {
        if (globalDescribeCache == null) {
            globalDescribeCache = Schema.getGlobalDescribe();
        }
        return globalDescribeCache.get(objectName.toLowerCase());
    }
}
```

**Regla:** Nunca llames `Schema.getGlobalDescribe()` en un loop ni en un trigger sin cacheo.

## Dynamic DML — Operar sobre SObjects desconocidos

```apex
// Crear un registro de cualquier tipo en runtime
String objectName = 'Contact'; // podría venir de config
SObject newRecord = Schema.getGlobalDescribe().get(objectName).newSObject();

newRecord.put('LastName', 'García');
newRecord.put('Email', 'garcia@example.com');

insert newRecord;
Id newId = newRecord.Id;
```

### DML con manejo de errores parciales

```apex
List<Database.SaveResult> results = Database.insert(records, false);

List<Integration_Error__c> errors = new List<Integration_Error__c>();
for (Integer i = 0; i < results.size(); i++) {
    if (!results[i].isSuccess()) {
        for (Database.Error err : results[i].getErrors()) {
            errors.add(new Integration_Error__c(
                Record_Id__c    = String.valueOf(records[i].get('Id')),
                Error_Code__c   = String.valueOf(err.getStatusCode()),
                Error_Message__c = err.getMessage(),
                Fields__c       = String.join(err.getFields(), ', ')
            ));
        }
    }
}
if (!errors.isEmpty()) insert errors;
```

## `Type.forName()` — Instanciar clases dinámicamente

El puente entre configuración y código. Permite que Custom Metadata decida qué clase ejecutar.

```apex
// Una interfaz que todos los handlers implementan
public interface IntegrationHandler {
    void execute(List<SObject> records);
}

// En runtime, decidir cuál usar por configuración
String handlerClass = 'ContactSyncHandler'; // viene de Custom Metadata
Type handlerType = Type.forName(handlerClass);

if (handlerType == null) {
    throw new IntegrationException('Handler no encontrado: ' + handlerClass);
}

IntegrationHandler handler = (IntegrationHandler) handlerType.newInstance();
handler.execute(records);
```

## Patrón arquitectónico: Mapper de Integración Genérico

El uso más poderoso de Dynamic Apex en integración — mapeo de campos configurable sin código:

```
Integration_Field_Map__mdt
├── Source_Field__c  = 'firstName'    (campo del JSON externo)
├── Target_Field__c  = 'FirstName'    (campo en Salesforce)
├── SObject__c       = 'Contact'
└── Transform__c     = 'UPPERCASE'
```

```apex
public class DynamicFieldMapper {

    public static SObject mapFromJson(String objectName, Map<String, Object> payload) {
        List<Integration_Field_Map__mdt> mappings = [
            SELECT Source_Field__c, Target_Field__c, Transform__c
            FROM Integration_Field_Map__mdt
            WHERE SObject__c = :objectName AND Is_Active__c = true
        ];

        Schema.SObjectType sObjType = SchemaCache.getSObjectType(objectName);
        if (sObjType == null) throw new MappingException('Objeto no encontrado: ' + objectName);

        SObject record = sObjType.newSObject();
        Map<String, Schema.SObjectField> fields = sObjType.getDescribe().fields.getMap();

        for (Integration_Field_Map__mdt mapping : mappings) {
            if (!payload.containsKey(mapping.Source_Field__c)) continue;

            Object value = payload.get(mapping.Source_Field__c);

            String targetField = mapping.Target_Field__c.toLowerCase();
            if (!fields.containsKey(targetField)) continue;
            if (!fields.get(targetField).getDescribe().isUpdateable()) continue;

            if (mapping.Transform__c == 'UPPERCASE' && value instanceof String) {
                value = ((String) value).toUpperCase();
            }

            record.put(mapping.Target_Field__c, value);
        }

        return record;
    }
}
```

Con esto, agregar un nuevo campo al mapeo es cambiar una fila en Custom Metadata — cero código, cero deploy.

## Ejercicio práctico

Construye un `GenericUpsertService` que recibe `String objectName`, `String externalIdField` y `List<Map<String, Object>>` con payloads:

1. Valida que el objeto existe usando Schema Describe
2. Valida que cada campo en el payload existe y es editable antes de setearlo
3. Construye los SObjects dinámicamente con `put()`
4. Hace upsert con el External ID field especificado
5. Maneja errores parciales con `Database.upsert(records, false)` y loggea fallos
6. El query para registros existentes usa `queryWithBinds` con `USER_MODE`

Tests deben cubrir: campo inexistente (ignorado), campo no editable (ignorado), registro con error (loggeado sin cancelar los demás).
