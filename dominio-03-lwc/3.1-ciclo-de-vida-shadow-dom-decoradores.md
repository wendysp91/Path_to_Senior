# 3.1 — Ciclo de Vida, Shadow DOM, Decoradores

---

## Ciclo de vida de un componente LWC

El ciclo de vida define el orden en que el framework crea, monta, actualiza y destruye un componente. Cada fase tiene un hook (callback) que puedes implementar.

### Orden de hooks

```
constructor()
    │
    ▼
connectedCallback()
    │
    ▼
[render — el framework pinta el template]
    │
    ▼
renderedCallback()       ← se llama cada vez que re-renderiza
    │
    ▼
[el componente vive y reacciona a cambios de propiedades]
    │
    ▼
disconnectedCallback()   ← cuando se elimina del DOM
    │
    ▼
errorCallback(error, stack)  ← cuando un hijo lanza error
```

### constructor()

- Se llama cuando se crea la instancia del componente, antes de que el template se pinte.
- Debes llamar `super()` como primera línea.
- No puedes acceder aún al template ni a los hijos (`this.template.querySelector` devuelve null).
- No puedes asignar propiedades públicas (`@api`) aquí — aún no están hidratadas.

```js
constructor() {
    super();
    this.privateValue = 'init'; // OK
}
```

### connectedCallback()

- Se llama cuando el componente se inserta en el DOM.
- Aquí ya puedes acceder a propiedades `@api` y hacer llamadas a servicios o subscribir a eventos.
- Se puede llamar múltiples veces si el componente se mueve en el DOM.

```js
connectedCallback() {
    this.loadData();
    this.subscription = subscribe(this.messageContext, CHANNEL, (msg) => this.handleMessage(msg));
}
```

### renderedCallback()

- Se llama después de cada render (incluido el primero y cada actualización posterior).
- Usar con cuidado: si modificas propiedades aquí puedes provocar un bucle infinito de renders.
- Patrón para ejecutar algo solo en el primer render:

```js
isFirstRender = true;

renderedCallback() {
    if (this.isFirstRender) {
        this.isFirstRender = false;
        // inicialización que requiere el DOM pintado
        this.initializeChart();
    }
}
```

### disconnectedCallback()

- Se llama cuando el componente se elimina del DOM.
- Ideal para limpiar: cancelar suscripciones, timers, event listeners manuales.

```js
disconnectedCallback() {
    unsubscribe(this.subscription);
    clearInterval(this.timer);
}
```

### errorCallback(error, stack)

- Solo disponible en componentes que actúan como **error boundary** (contenedores de error).
- Captura errores de componentes hijo durante el ciclo de vida.
- Útil para mostrar UI de fallback en lugar de romper toda la página.

```js
errorCallback(error, stack) {
    this.errorMessage = error.message;
}
```

### Orden en árbol padre-hijo

```
Padre: constructor → connectedCallback
    Hijo: constructor → connectedCallback
    Hijo: renderedCallback
Padre: renderedCallback
```

Los hijos se montan completamente antes de que el padre llame a `renderedCallback`.

## Shadow DOM

LWC usa Shadow DOM para encapsular el CSS y el DOM interno de cada componente.

### Qué encapsula

- **CSS:** los estilos definidos en `component.css` solo aplican a ese componente. No se "filtran" hacia afuera ni hacia adentro.
- **DOM:** desde fuera del componente no puedes hacer `document.querySelector` y alcanzar elementos dentro del shadow.

### Acceso desde dentro del componente

```js
// Correcto — accede al template del propio componente
this.template.querySelector('input');
this.template.querySelectorAll('.item');

// Incorrecto — document no puede ver el shadow
document.querySelector('input'); // null en la mayoría de casos
```

### CSS personalizable: Custom Properties (CSS Variables)

La única forma de que estilos externos afecten al interior de un componente es a través de **CSS Custom Properties**:

```css
/* componente hijo — usa la variable */
.container {
    background-color: var(--brand-color, #fff);
}

/* página o componente padre — define la variable */
c-mi-hijo {
    --brand-color: #0070d2;
}
```

### Slots

Permiten que el padre inserte contenido dentro del template del hijo.

```html
<!-- hijo: my-card.html -->
<template>
    <div class="card">
        <slot></slot>              <!-- slot por defecto -->
        <slot name="footer"></slot> <!-- slot con nombre -->
    </div>
</template>

<!-- padre -->
<c-my-card>
    <p>Contenido principal</p>
    <span slot="footer">Pie de tarjeta</span>
</c-my-card>
```

## Decoradores

Los decoradores son marcadores que le dicen al framework cómo tratar una propiedad o método.

### @api — propiedad pública

Expone una propiedad o método para que el padre pueda pasarle valores o llamarlo.

```js
import { LightningElement, api } from 'lwc';

export default class MyComponent extends LightningElement {
    @api title;           // el padre puede pasar: <c-my-component title="Hola">
    @api recordId;        // automáticamente se llena en páginas de registro

    @api
    resetForm() {         // el padre puede llamar: this.template.querySelector('c-my-component').resetForm()
        this.value = '';
    }
}
```

**Reglas de `@api`:**
- Las propiedades `@api` son **inmutables** desde dentro del componente — no puedes hacer `this.title = 'nuevo'` directamente; si necesitas mutarla internamente, copia el valor a una propiedad privada.
- Los primitivos se pasan por valor, los objetos/arrays por referencia (clonar si necesitas mutarlos).

### @track — reactividad profunda (legacy)

Antes de la versión Spring '20, se necesitaba `@track` para que el framework detectara cambios en propiedades de objetos o arrays. Hoy **todas las propiedades son reactivas** por defecto para primitivos. Para objetos/arrays, `@track` sigue siendo necesario si quieres reactividad profunda (cambios en propiedades anidadas).

```js
// Sin @track: el template se actualiza si 'user' cambia de referencia completa
user = { name: 'Ana', age: 30 };

// Con @track: el template se actualiza si cambia user.name o user.age
@track user = { name: 'Ana', age: 30 };
```

En la práctica moderna: si mutes el objeto completo (`this.user = { ...this.user, name: 'Carlos' }`) no necesitas `@track`. Si mutes solo una propiedad interna (`this.user.name = 'Carlos'`), sí necesitas `@track`.

### @wire — conexión reactiva a datos

Conecta una propiedad o función a un Wire Adapter (Apex, UI API, etc.). Se cubre en detalle en el tema 3.3.

```js
@wire(getRecord, { recordId: '$recordId', fields: FIELDS })
record;
```

## Resumen rápido

| Concepto | Clave |
|---|---|
| `constructor` | Antes del DOM; siempre `super()` primero; sin acceso a `@api` |
| `connectedCallback` | Al insertar en DOM; suscripciones, llamadas iniciales |
| `renderedCallback` | Después de cada render; cuidado con bucles; usar flag para "first render" |
| `disconnectedCallback` | Al salir del DOM; limpiar suscripciones y timers |
| `errorCallback` | Error boundary para errores de componentes hijo |
| Shadow DOM | CSS y DOM encapsulados; acceso vía `this.template.querySelector` |
| CSS Custom Properties | La única forma de estilar desde fuera del shadow |
| `@api` | Propiedad/método público; inmutable desde adentro |
| `@track` | Reactividad profunda para objetos/arrays mutados internamente |
