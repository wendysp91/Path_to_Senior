# 3.2 — Comunicación entre Componentes

---

## Patrones de comunicación

| Dirección | Mecanismo |
|---|---|
| Padre → Hijo | Propiedades `@api` |
| Hijo → Padre | Eventos (`CustomEvent`) |
| Componentes sin relación directa | Lightning Message Service (LMS) |
| Publicar/suscribir global | LMS o librería pub/sub custom |

---

## Padre → Hijo: Propiedades @api

El padre pasa datos al hijo a través de atributos en el template. El hijo los recibe como propiedades `@api`.

```html
<!-- padre.html -->
<template>
    <c-hijo titulo="Hola" record-id={currentId}></c-hijo>
</template>
```

```js
// hijo.js
import { LightningElement, api } from 'lwc';
export default class Hijo extends LightningElement {
    @api titulo;
    @api recordId;  // camelCase en JS, kebab-case en HTML
}
```

**Regla de naming:** `camelCase` en JS → `kebab-case` en HTML.  
`recordId` en JS = `record-id` en el template del padre.

### Llamar métodos del hijo desde el padre

```html
<!-- padre.html -->
<template>
    <c-hijo lwc:ref="miHijo"></c-hijo>
    <button onclick={handleClick}>Reset</button>
</template>
```

```js
// padre.js
handleClick() {
    this.refs.miHijo.resetForm(); // llama método @api del hijo
}
```

> `lwc:ref` está disponible desde la API versión 59+. Para versiones anteriores se usaba `this.template.querySelector('c-hijo')`.

## Hijo → Padre: CustomEvent

El hijo dispara un evento; el padre lo escucha con un handler.

### En el hijo — disparar el evento

```js
// hijo.js
handleSave() {
    const event = new CustomEvent('save', {
        detail: { id: this.recordId, name: this.name },
        bubbles: true,    // ¿el evento sube por el DOM?
        composed: false   // ¿el evento cruza el shadow boundary?
    });
    this.dispatchEvent(event);
}
```

### En el padre — escuchar el evento

```html
<!-- padre.html -->
<template>
    <c-hijo onsave={handleSave}></c-hijo>
</template>
```

```js
// padre.js
handleSave(event) {
    const { id, name } = event.detail;
    console.log('Guardado:', id, name);
}
```

### bubbles y composed

| Propiedad | Valor | Significado |
|---|---|---|
| `bubbles` | `false` (default) | El evento solo lo escucha el padre directo |
| `bubbles` | `true` | El evento sube por el árbol DOM |
| `composed` | `false` (default) | El evento NO cruza el shadow boundary |
| `composed` | `true` | El evento cruza shadows (necesario para escucharlo en ancestros lejanos) |

**Regla general:** usa `bubbles: false, composed: false` (comunicación directa padre-hijo). Solo activa `bubbles` y `composed` si necesitas que un ancestro lejano escuche el evento, pero evítalo por acoplamiento.

## Componentes sin relación directa: Lightning Message Service (LMS)

Permite comunicación entre componentes que no están en la misma jerarquía de DOM — por ejemplo, dos componentes en regiones distintas de una Lightning Page, o un Aura y un LWC.

### 1. Definir el Message Channel

Se define en metadata como un archivo `.messageChannel-meta.xml`:

```xml
<!-- force-app/main/default/messageChannels/OrderUpdate.messageChannel-meta.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<LightningMessageChannel xmlns="http://soap.sforce.com/2006/04/metadata">
    <masterLabel>OrderUpdate</masterLabel>
    <isExposed>true</isExposed>
    <fields>
        <fieldName>orderId</fieldName>
        <description>ID of the updated order</description>
    </fields>
    <fields>
        <fieldName>status</fieldName>
    </fields>
</LightningMessageChannel>
```

### 2. Publicar (Publisher)

```js
import { LightningElement, wire } from 'lwc';
import { MessageContext, publish } from 'lightning/messageService';
import ORDER_UPDATE from '@salesforce/messageChannel/OrderUpdate__c';

export default class Publisher extends LightningElement {
    @wire(MessageContext) messageContext;

    handleUpdate() {
        publish(this.messageContext, ORDER_UPDATE, {
            orderId: '001...',
            status: 'Shipped'
        });
    }
}
```

### 3. Suscribir (Subscriber)

```js
import { LightningElement, wire } from 'lwc';
import { MessageContext, subscribe, unsubscribe, APPLICATION_SCOPE } from 'lightning/messageService';
import ORDER_UPDATE from '@salesforce/messageChannel/OrderUpdate__c';

export default class Subscriber extends LightningElement {
    @wire(MessageContext) messageContext;
    subscription = null;

    connectedCallback() {
        this.subscription = subscribe(
            this.messageContext,
            ORDER_UPDATE,
            (message) => this.handleMessage(message),
            { scope: APPLICATION_SCOPE }  // escucha aunque el componente no sea visible
        );
    }

    disconnectedCallback() {
        unsubscribe(this.subscription);
        this.subscription = null;
    }

    handleMessage(message) {
        console.log('Order:', message.orderId, message.status);
    }
}
```

### Scopes de LMS

| Scope | Descripción |
|---|---|
| `DEFAULT_SCOPE` | Solo escucha si el componente está activo en la página actual |
| `APPLICATION_SCOPE` | Escucha siempre, incluso desde tabs inactivas |

## Acceso directo entre componentes: lwc:ref y querySelector

Para acceder a métodos públicos de un hijo desde el padre:

```js
// Con lwc:ref (API 59+)
this.refs.miHijo.resetForm();

// Con querySelector
this.template.querySelector('c-mi-hijo').resetForm();
```

No puedes acceder a propiedades o métodos privados del hijo. Solo los marcados con `@api`.

---

## Comparativa de patrones

| Patrón | Uso | Acoplamiento |
|---|---|---|
| `@api` | Padre pasa datos al hijo | Alto (conocen su relación) |
| `CustomEvent` | Hijo notifica al padre | Medio |
| LMS | Cualquier componente a cualquier componente | Bajo (desacoplado) |
| `querySelector` + `@api method` | Padre llama método del hijo | Alto |

---

## Resumen rápido

| Concepto | Clave |
|---|---|
| Padre → Hijo | Atributos `@api`; camelCase en JS = kebab-case en HTML |
| Hijo → Padre | `CustomEvent` + `dispatchEvent`; padre escucha con `on<eventname>` |
| `bubbles/composed` | Default false/false; solo activar si el evento necesita subir por el árbol |
| LMS | Para componentes sin jerarquía común; requiere Message Channel en metadata |
| `APPLICATION_SCOPE` | Suscripción que sobrevive aunque el componente no sea visible |
| `lwc:ref` | Acceso tipado al hijo desde el padre (API 59+) |
