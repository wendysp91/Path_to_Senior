# 3.4 — Navegación, Performance y Testing con Jest

---

## Navegación

### NavigationMixin

El mixin estándar para navegar entre páginas en Salesforce desde un LWC.

```js
import { LightningElement } from 'lwc';
import { NavigationMixin } from 'lightning/navigation';

export default class MyComponent extends NavigationMixin(LightningElement) {

    navigateToRecord() {
        this[NavigationMixin.Navigate]({
            type: 'standard__recordPage',
            attributes: {
                recordId: this.recordId,
                actionName: 'view'  // 'view' | 'edit' | 'clone'
            }
        });
    }
}
```

### Page References más comunes

| Tipo | Uso |
|---|---|
| `standard__recordPage` | Vista/edición de un registro |
| `standard__objectPage` | List view de un objeto |
| `standard__namedPage` | Páginas nombradas (Home, Chatter, etc.) |
| `standard__component` | Abrir un LWC como página |
| `standard__webPage` | URL externa |
| `comm__namedPage` | Página de Experience Cloud |

```js
// List view
this[NavigationMixin.Navigate]({
    type: 'standard__objectPage',
    attributes: { objectApiName: 'Account', actionName: 'list' },
    state: { filterName: 'All' }
});

// URL externa
this[NavigationMixin.Navigate]({
    type: 'standard__webPage',
    attributes: { url: 'https://example.com' }
});

// Obtener URL sin navegar (útil para <a href>)
this[NavigationMixin.GenerateUrl]({ type: 'standard__recordPage', ... })
    .then(url => { this.recordUrl = url; });
```

## Performance

### Renderizado condicional: if:true vs lwc:if

Desde API versión 58+, se reemplaza `if:true` / `if:false` por `lwc:if`, `lwc:elseif`, `lwc:else`:

```html
<!-- Legacy (aún funciona) -->
<template if:true={isLoading}>
    <lightning-spinner></lightning-spinner>
</template>

<!-- Moderno (API 58+) -->
<template lwc:if={isLoading}>
    <lightning-spinner></lightning-spinner>
</template>
<template lwc:elseif={hasError}>
    <p>{errorMessage}</p>
</template>
<template lwc:else>
    <c-data-table records={data}></c-data-table>
</template>
```

`lwc:if` crea y destruye el componente del DOM. Para ocultar sin destruir usa CSS (`display:none`) cuando el costo de recrear el componente es alto.

### for:each vs iterator

```html
<!-- for:each — el más común -->
<template for:each={items} for:item="item">
    <p key={item.Id}>{item.Name}</p>
</template>

<!-- iterator — cuando necesitas saber si es primero/último -->
<template iterator:it={items}>
    <li key={it.value.Id}
        class={it.first ? 'first' : ''}>{it.value.Name}</li>
</template>
```

**`key` es obligatorio** en listas — debe ser único y estable (usa `Id`, no el índice del array).

### Lazy loading con dynamic import

Para cargar componentes pesados solo cuando se necesitan:

```js
async handleShowModal() {
    const { default: MyModal } = await import('c/myModal');
    // componente cargado solo al hacer clic
}
```

### Evitar renders innecesarios

- No mutes propiedades en `renderedCallback` sin una bandera de control.
- Para objetos `@track`, clona antes de mutar: `this.data = { ...this.data, field: value }`.
- Usa `getter` computados en lugar de recalcular en cada método.

```js
// Bien: se recalcula solo si la propiedad reactiva cambia
get filteredItems() {
    return this.items.filter(i => i.active);
}
```

### Platform Cache desde LWC

Para datos que no cambian frecuentemente, puedes usar Apex con `cacheable=true` — el browser cachea el resultado. Para caché más granular usa `@wire(getRecord)` con la UI API que tiene su propio cache interno.

## Testing con Jest

LWC usa Jest como framework de pruebas. Las pruebas viven en `__tests__/` dentro del directorio del componente.

```
force-app/main/default/lwc/
└── myComponent/
    ├── myComponent.html
    ├── myComponent.js
    ├── myComponent.css
    └── __tests__/
        └── myComponent.test.js
```

### Setup básico

```bash
npm install @salesforce/sfdx-lwc-jest --save-dev
```

`package.json`:
```json
{
  "scripts": {
    "test:unit": "lwc-jest"
  }
}
```

### Estructura de un test

```js
import { createElement } from 'lwc';
import MyComponent from 'c/myComponent';

describe('c-my-component', () => {

    afterEach(() => {
        while (document.body.firstChild) {
            document.body.removeChild(document.body.firstChild);
        }
    });

    it('renders the title correctly', () => {
        const element = createElement('c-my-component', { is: MyComponent });
        element.title = 'Test Title';
        document.body.appendChild(element);

        const p = element.shadowRoot.querySelector('p');
        expect(p.textContent).toBe('Test Title');
    });
});
```

**Puntos clave:**
- `createElement` crea la instancia pero no la monta.
- `document.body.appendChild(element)` la monta — aquí se ejecutan `constructor` y `connectedCallback`.
- Acceso al DOM interno: `element.shadowRoot.querySelector()`.
- Limpiar el DOM en `afterEach` para evitar contaminación entre tests.

### Mockear módulos de Salesforce

Jest no tiene acceso a Salesforce, así que debes mockear los imports:

```js
// Mockear una llamada Apex
import getAccounts from '@salesforce/apex/AccountController.getAccounts';

jest.mock(
    '@salesforce/apex/AccountController.getAccounts',
    () => ({ default: jest.fn() }),
    { virtual: true }
);

it('loads accounts on wire', async () => {
    getAccounts.mockResolvedValue([{ Id: '001', Name: 'Acme' }]);

    const element = createElement('c-account-list', { is: AccountList });
    document.body.appendChild(element);

    await Promise.resolve(); // espera un ciclo de render

    const items = element.shadowRoot.querySelectorAll('p');
    expect(items.length).toBe(1);
});
```

### Mockear @wire

El paquete `@salesforce/sfdx-lwc-jest` incluye utilidades para mockear wire:

```js
import { registerApexTestWireAdapter } from '@salesforce/sfdx-lwc-jest';
import getAccounts from '@salesforce/apex/AccountController.getAccounts';

const getAccountsAdapter = registerApexTestWireAdapter(getAccounts);

it('renders data from wire', async () => {
    const element = createElement('c-account-list', { is: AccountList });
    document.body.appendChild(element);

    getAccountsAdapter.emit({ data: [{ Id: '001', Name: 'Acme' }], error: undefined });
    await Promise.resolve();

    const p = element.shadowRoot.querySelector('p');
    expect(p.textContent).toBe('Acme');
});
```

### Probar eventos

```js
it('dispatches save event', async () => {
    const element = createElement('c-my-form', { is: MyForm });
    document.body.appendChild(element);

    const handler = jest.fn();
    element.addEventListener('save', handler);

    element.shadowRoot.querySelector('button').click();
    await Promise.resolve();

    expect(handler).toHaveBeenCalledTimes(1);
    expect(handler.mock.calls[0][0].detail.name).toBe('Test');
});
```

## Resumen rápido

| Concepto | Clave |
|---|---|
| `NavigationMixin` | Extiende el componente; navega con `this[NavigationMixin.Navigate]({...})` |
| `standard__recordPage` | Navegar a vista/edición de un registro |
| `lwc:if/elseif/else` | Sustituto moderno de `if:true` (API 58+) |
| `key` en listas | Obligatorio, único y estable (usa `Id`) |
| Jest setup | `createElement` → `appendChild` → query `shadowRoot` |
| Mock de Apex en Jest | `jest.mock` con `{ virtual: true }` |
| Mock de `@wire` | `registerApexTestWireAdapter` + `.emit({data, error})` |
| `afterEach` cleanup | Remover elementos del DOM entre tests |
