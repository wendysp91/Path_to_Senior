# 3.3 — Wire Service e Integración con Apex

---

## ¿Qué es el Wire Service?

El Wire Service es el mecanismo reactivo de LWC para obtener datos declarativamente. Cuando los parámetros cambian, el framework re-ejecuta la llamada automáticamente y actualiza el componente.

Funciona con **Wire Adapters** — cualquier función decorada con `@wire` que el framework sabe cómo invocar de forma reactiva.

---

## @wire con Apex

### Preparar el método Apex

El método debe ser:
- `@AuraEnabled(cacheable=true)` — obligatorio para usarlo con `@wire`.
- `static` y `public` o `global`.

```apex
public with sharing class AccountController {
    @AuraEnabled(cacheable=true)
    public static List<Account> getAccounts(String industry) {
        return [SELECT Id, Name, Industry FROM Account WHERE Industry = :industry LIMIT 50];
    }
}
```

> `cacheable=true` permite que el resultado se almacene en caché del cliente. Sin él, el método solo puede llamarse con `imperative call`.

### Importar y usar con @wire

```js
import { LightningElement, api, wire } from 'lwc';
import getAccounts from '@salesforce/apex/AccountController.getAccounts';

export default class AccountList extends LightningElement {
    @api industry = 'Technology';

    @wire(getAccounts, { industry: '$industry' })
    accounts;
    // accounts = { data: [...], error: undefined } o { data: undefined, error: {...} }
}
```

### Parámetros reactivos

El prefijo `$` hace que el parámetro sea **reactivo**: si `this.industry` cambia, el wire se re-ejecuta automáticamente.

```js
// Sin $: valor estático, no re-ejecuta si cambia
@wire(getAccounts, { industry: 'Technology' })

// Con $: reactivo, re-ejecuta cuando this.industry cambia
@wire(getAccounts, { industry: '$industry' })
```

### Dos formas de recibir el resultado

**Forma 1 — Propiedad (objeto con data/error):**

```js
@wire(getAccounts, { industry: '$industry' })
accounts; // { data, error }
```

```html
<template>
    <template if:true={accounts.data}>
        <template for:each={accounts.data} for:item="acc">
            <p key={acc.Id}>{acc.Name}</p>
        </template>
    </template>
    <template if:true={accounts.error}>
        <p>Error: {accounts.error.body.message}</p>
    </template>
</template>
```

**Forma 2 — Función (más control):**

```js
@wire(getAccounts, { industry: '$industry' })
wiredAccounts({ data, error }) {
    if (data) {
        this.accounts = data;
        this.error = undefined;
    } else if (error) {
        this.error = error;
        this.accounts = undefined;
    }
}
```

## @wire con UI API (Wire Adapters estándar)

Salesforce provee Wire Adapters para acceder a metadatos y datos sin escribir Apex.

### getRecord — obtener un registro

```js
import { LightningElement, api, wire } from 'lwc';
import { getRecord, getFieldValue } from 'lightning/uiRecordApi';
import NAME_FIELD from '@salesforce/schema/Account.Name';
import INDUSTRY_FIELD from '@salesforce/schema/Account.Industry';

const FIELDS = [NAME_FIELD, INDUSTRY_FIELD];

export default class AccountDetail extends LightningElement {
    @api recordId;

    @wire(getRecord, { recordId: '$recordId', fields: FIELDS })
    account;

    get name() {
        return getFieldValue(this.account.data, NAME_FIELD);
    }
}
```

> Usa `getFieldValue` para extraer el valor de un campo del objeto `record` de la UI API — no accedas directamente al JSON interno.

### Otros Wire Adapters comunes

| Adapter | Importar desde | Uso |
|---|---|---|
| `getRecord` | `lightning/uiRecordApi` | Obtener un registro por Id |
| `getListUi` | `lightning/uiListApi` | Obtener una list view |
| `getPicklistValues` | `lightning/uiObjectInfoApi` | Valores de un campo picklist |
| `getObjectInfo` | `lightning/uiObjectInfoApi` | Metadatos del objeto |
| `getRelatedListRecords` | `lightning/uiRelatedListApi` | Registros de una lista relacionada |
| `MessageContext` | `lightning/messageService` | Contexto para LMS |

## Llamadas imperativas (sin @wire)

Para casos donde necesitas control total: llamar Apex bajo demanda, en respuesta a un evento, o cuando el resultado NO debe cachearse.

```js
import { LightningElement } from 'lwc';
import saveAccount from '@salesforce/apex/AccountController.saveAccount';

export default class AccountForm extends LightningElement {
    handleSave() {
        saveAccount({ name: this.name, industry: this.industry })
            .then(result => {
                this.recordId = result;
                this.showToast('Guardado', 'success');
            })
            .catch(error => {
                this.errorMessage = error.body.message;
            });
    }
}
```

El método Apex puede ser `cacheable=false` (o sin el atributo, que es el default) para llamadas imperativas que modifican datos (insert/update/delete).

### @wire vs Llamada imperativa

| Aspecto | @wire | Imperativa |
|---|---|---|
| Cuándo se ejecuta | Reactivo — automático al cambiar parámetros | Manualmente en código |
| Cacheable | Requiere `cacheable=true` | Funciona con y sin cache |
| DML (insert/update/delete) | NO — solo lectura con cacheable | SÍ |
| Retry manual | No (se re-ejecuta por reactividad) | Sí, puedes llamarlo cuando quieras |
| Uso ideal | Lectura de datos en carga del componente | Acciones del usuario, mutaciones de datos |

## Actualizar datos en caché con refreshApex

Cuando usas `@wire` con un método `cacheable=true`, el resultado queda en caché. Después de una mutación de datos, necesitas forzar la actualización:

```js
import { LightningElement, wire } from 'lwc';
import { refreshApex } from '@salesforce/apex';
import getAccounts from '@salesforce/apex/AccountController.getAccounts';

export default class AccountList extends LightningElement {
    @wire(getAccounts, { industry: '$industry' })
    wiredResult; // guardar la referencia del wire para refreshApex

    handleRefresh() {
        refreshApex(this.wiredResult); // re-llama el método y actualiza el componente
    }
}
```

> `refreshApex` solo funciona si guardas la referencia completa del wire (no el `data` extraído).

## Resumen rápido

| Concepto | Clave |
|---|---|
| `@AuraEnabled(cacheable=true)` | Obligatorio para usar Apex con `@wire` |
| Parámetro `$` | Prefijo para hacer un parámetro reactivo en `@wire` |
| Resultado del wire | `{ data, error }` — siempre valida ambos |
| Wire como función | Más control; se ejecuta cada vez que el wire resuelve |
| `getRecord` + `getFieldValue` | Patrón estándar para leer registros sin Apex |
| Llamada imperativa | Para DML o llamadas bajo demanda; Apex sin `cacheable=true` |
| `refreshApex` | Invalida la caché del wire y re-ejecuta la llamada |
