# 2.4 — SOSL (Salesforce Object Search Language)

---

## ¿Qué es SOSL?

Lenguaje de búsqueda de texto completo de Salesforce. Busca un término en múltiples objetos y múltiples campos a la vez, usando el motor de búsqueda interno (search index), no la base de datos relacional.

### SOQL vs SOSL

| Aspecto | SOQL | SOSL |
|---|---|---|
| Motor | Base de datos (SQL-like) | Motor de búsqueda de texto |
| Busca en | Un objeto a la vez | Múltiples objetos a la vez |
| Filtro principal | Condiciones exactas (`=`, `>`, etc.) | Término de texto (parcial o exacto) |
| Devuelve | `List<SObject>` | `List<List<SObject>>` |
| Null values | Sí los encuentra | No los indexa |
| Uso ideal | Queries estructurados con condiciones | Búsqueda tipo "search box" multi-objeto |

---

## Sintaxis básica

```sosl
FIND {término} IN SearchGroup RETURNING ObjectType1(fields), ObjectType2(fields)
```

```sosl
FIND {Acme} IN ALL FIELDS RETURNING Account(Id, Name), Contact(Id, LastName, Email)
```

### Componentes

| Componente | Descripción |
|---|---|
| `FIND {término}` | Término de búsqueda. Va entre llaves `{}` |
| `IN SearchGroup` | Qué campos buscar (ver abajo) |
| `RETURNING` | Objetos y campos a devolver |

---

## Search Groups

Define en qué tipo de campos busca:

| Search Group | Busca en |
|---|---|
| `ALL FIELDS` | Todos los campos de texto, email, phone, etc. (default) |
| `NAME FIELDS` | Solo campos de nombre (Name, FirstName, LastName) |
| `EMAIL FIELDS` | Solo campos de tipo Email |
| `PHONE FIELDS` | Solo campos de tipo Phone |
| `SIDEBAR FIELDS` | Campos de la barra lateral (Legacy, poco usado) |

```sosl
FIND {maria} IN EMAIL FIELDS RETURNING Contact(Id, LastName, Email)
FIND {Acme} IN NAME FIELDS RETURNING Account(Id, Name), Lead(Id, Company)
```

---

## Operadores de búsqueda

### Comodines

| Operador | Significado | Ejemplo |
|---|---|---|
| `*` | Cero o más caracteres (al final) | `Acm*` → Acme, Acmex, Acme Corp |
| `?` | Exactamente un carácter | `Ac?e` → Acme, Acne |

> Los comodines solo van al **final o en medio** del término, nunca al inicio.

### Operadores lógicos

```sosl
FIND {Acme AND Corp}          -- ambos términos presentes
FIND {Acme OR Corp}           -- cualquiera de los dos
FIND {Acme NOT Corp}          -- Acme pero no Corp
FIND {"Acme Corp"}            -- frase exacta (comillas dobles)
```

### Combinaciones

```sosl
FIND {(Acme OR Corp) AND Mexico}
```

---

## RETURNING — filtrar y ordenar resultados

Dentro del `RETURNING` puedes agregar condiciones `WHERE`, `ORDER BY` y `LIMIT` por objeto:

```sosl
FIND {Acme} IN ALL FIELDS
RETURNING
    Account(Id, Name WHERE Industry = 'Technology' ORDER BY Name LIMIT 10),
    Contact(Id, LastName, Email WHERE AccountId != null)
```

### LIMIT global vs LIMIT por objeto

```sosl
-- LIMIT global: máximo de registros totales devueltos (default 2000, max 2000)
FIND {Acme} IN ALL FIELDS RETURNING Account(Id, Name), Contact(Id, LastName) LIMIT 200

-- LIMIT por objeto: dentro del RETURNING
FIND {Acme} IN ALL FIELDS RETURNING Account(Id, Name LIMIT 50), Contact(Id, LastName LIMIT 50)
```

## Usar SOSL en Apex

### Search.query() — resultado tipado como List<List<SObject>>

```apex
List<List<SObject>> results = [FIND {Acme} IN ALL FIELDS
                                RETURNING Account(Id, Name), Contact(Id, LastName)];

List<Account> accounts = (List<Account>) results[0];
List<Contact> contacts = (List<Contact>) results[1];

for (Account acc : accounts) {
    System.debug(acc.Name);
}
```

El índice de la lista externa corresponde al **orden en que aparecen los objetos en RETURNING** (0 = primero, 1 = segundo, etc.).

### Search.query() con variable dinámica

No puedes interpolrar variables directamente en SOSL estático como en SOQL. Para búsquedas dinámicas usa `Search.query()`:

```apex
String term = 'Acme*';
String sosl = 'FIND {' + term + '} IN ALL FIELDS RETURNING Account(Id, Name), Contact(Id, LastName)';
List<List<SObject>> results = Search.query(sosl);
```

> Cuidado con injection: valida y escapa el término antes de concatenarlo.  
> Usa `String.escapeSingleQuotes()` no aplica aquí — para SOSL usa `Search.escape()` si el término viene del usuario.

```apex
String safeTerm = Search.escape(userInput);
String sosl = 'FIND {' + safeTerm + '} IN ALL FIELDS RETURNING Account(Id, Name)';
```

## Governor Limits de SOSL

| Límite | Valor |
|---|---|
| SOSL queries por transacción | 20 |
| Registros devueltos por query | 2,000 (máximo) |
| Registros por objeto en RETURNING | 200 (default), configurable con LIMIT hasta 2,000 |

---

## Cuándo usar SOSL vs SOQL

| Escenario | Usar |
|---|---|
| Buscar un término en múltiples objetos | SOSL |
| Search box o barra de búsqueda global | SOSL |
| Filtrar por condiciones exactas (fechas, Ids, status) | SOQL |
| Necesitas aggregate functions o GROUP BY | SOQL |
| Necesitas `FOR UPDATE` | SOQL |
| El campo puede ser null y necesitas encontrarlo | SOQL (SOSL no indexa nulos) |
| Query en un solo objeto con filtros precisos | SOQL |

## SOSL en Test Classes

En tests, `SOSL` no busca en el search index real. Debes registrar los datos esperados con `Test.setFixedSearchResults()`:

```apex
@isTest
static void testSearch() {
    Account acc = new Account(Name = 'Acme Corp');
    insert acc;

    // Registrar qué debe devolver el SOSL en el test
    Test.setFixedSearchResults(new List<Id>{ acc.Id });

    List<List<SObject>> results = [FIND {Acme} IN ALL FIELDS RETURNING Account(Id, Name)];
    List<Account> found = (List<Account>) results[0];

    Assert.areEqual(1, found.size());
    Assert.areEqual('Acme Corp', found[0].Name);
}
```

Sin `Test.setFixedSearchResults()`, el SOSL devuelve lista vacía aunque los registros existan.

## Resumen rápido

| Concepto | Clave |
|---|---|
| SOSL vs SOQL | SOSL = multi-objeto, texto. SOQL = un objeto, condiciones exactas |
| Sintaxis | `FIND {término} IN SearchGroup RETURNING Obj(fields)` |
| Resultado en Apex | `List<List<SObject>>` — índice = orden en RETURNING |
| Comodines | `*` al final, `?` un carácter. Nunca al inicio |
| Frase exacta | `{"Acme Corp"}` con comillas dobles dentro de las llaves |
| Governor limit | 20 SOSL por transacción, 2,000 registros máx |
| Tests | Obligatorio `Test.setFixedSearchResults()` o devuelve vacío |
| SOSL dinámico | `Search.query(sosl)` + `Search.escape(userInput)` |
